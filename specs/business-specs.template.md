# Specification

project title: ...

### 1. Overview

Summarize the goal and purpose of the project.
Example: *This system will automate the scheduling of maintenance tasks for industrial equipment using a web-based platform.*

### 2. Objectives

- List specific, measurable goals.
- Describe expected value or outcomes.

### 3. Scope

Define what is included and excluded from the project.
Example:

- In scope: Web dashboard, notifications, user authentication.
- Out of scope: Mobile app, advanced analytics.

### 4. Glossary

Define here the most useful vocabulary of the business domain. Those defintions can be different from the standard definition because those word are ubiquitous language applied to the modelisation for this sofware.

### 5. Functional Requirements

List what the system must do. Each requirement should be clear and testable.
Example:

- The system must allow users to log in via email and password.
- The system must generate daily usage reports.


### 6. Non-Functional Requirements

Define quality attributes such as performance, scalability, and usability.
Example:

- The system should handle 1,000 concurrent users.
- Response time must not exceed 2 seconds.


### 7. System Architecture

Provide a high-level view of the system architecture, including components, integrations, and data flow diagrams.

### 8. Technology Stack

List all chosen technologies, frameworks, and tools.
Example:

- Frontend: React
- Backend: Node.js, Express
- Database: PostgreSQL
- Hosting: AWS EC2
